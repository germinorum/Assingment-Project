Section 8: Windows Deployment Services (WDS)

Section 8: Windows Deployment Services (WDS)

Goals

  • Deploy an Operating System image to a blank VM client using WDS
  • Create a customized image using sysprep and WDS capture images

Implementation Steps

Step 1: Ensure AD DS, DHCP, and DNS Roles are Working

Before setting up WDS, confirm that your Active Directory, DHCP, and DNS server roles are working correctly on WS1, as they are critical for WDS functionality.

Step 2: Extract Boot and Install WIM Files from Windows 10 ISO

Mount the Windows 10 ISO and copy the boot.wim and install.wim files from the "Sources" folder to your local drive.

Step 3: Extract Install WIM File from Hyper-V Server Core ISO

Mount the Hyper-V Server Core ISO and extract the install.wim file to your local drive for use in deploying server images.

Step 4: Add WDS Role to WS1

Using Server Manager, add the WDS role by selecting both Deployment Server and Transport Server options.

Step 5: Use WDS Manager to Add Boot and Install Images

Use WDS Manager to add boot and install images, organizing them into image groups for easier management.

Step 6: Create a New Client VM and Boot via PXE

Create a new VM, connect it to the network, and use PXE boot to load the boot image from WDS. Select the desired Windows 10 install image.

- Ensure the VM receives a DHCP address from the network.
- Press F12 to start PXE boot.
Step 7: Create a Customized Image using Sysprep

After customizing the VM, run Sysprep to generalize the OS. Use WDS to capture the image, and reboot the client to upload the image to the WDS server.

Step 8: Deploy the Customized Image

Reboot the VM, perform another PXE boot, and select the original boot image to deploy the customized image.
