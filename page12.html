Section 9: Managing VMs

Section 9: Managing VMs

Goals

  • Create a Linux virtual machine (VM) using Hyper-V manager
  • Create a VM Checkpoint
  • Replicate the VM
  • Configure and perform a Live Migration of a VM
  • Create a VM template using Sys prep utility

Implementation Steps

Step 1: Ensure Virtualization Extensions are Enabled

Ensure your Virtual Machine settings have Intel-VT or AMD-V Virtualization extensions enabled.

Step 2: Install Hyper-V Role on Both Servers

Use Server Manager to install the Hyper-V server role on both servers.

üí° Make sure to link to the physical network adapter.

Step 3: Reset Manual IP Addresses if Needed

If the IP addresses of the servers were set manually, they may need to be reset after installing the Hyper-V Manager.

Step 4: Create an External Virtual Switch

Create an external virtual switch that is bridged to the physical network card using Hyper-V Manager. If this is not done by default, you will need to configure it manually.

Step 5: Create a Generation 2 VM Using Hyper-V Manager

Create a Generation 2 virtual machine using Hyper-V Manager.

- Connect it to your external virtual switch.
- Use a Linux ISO file (free download), such as Tiny Core or Ubuntu, to install the OS on the VM.
Step 6: Create a VM Checkpoint

Make some changes on the Linux VM, and then create a VM Checkpoint using Hyper-V Manager.

Step 7: Replicate the VM on to the Other Server (WS1)

Use Hyper-V Manager to replicate the VM from WS2 to WS1.

- Enable WS2 as a Replica server in the Hyper-V settings of the VM.
- Use Kerberos (HTTP) authentication on port 80.
- Specify a default path to store the replica files.
On WS1:
  • Select "Enable Replication" on the VM in Hyper-V Manager.
  • Choose WS2 as the replica server.
  • Specify a replication frequency of 15 minutes.
  • Maintain only the latest recovery point (24 hours).
  • Send the initial copy over the network.
Step 8: Perform a Live Migration of a VM

Configure and perform a live migration of the VM from WS2 to WS1.

- Start the VM on WS2.
- Add the Microsoft Virtual System Migration Service to the AD computer properties of WS2.
- Enable incoming and outgoing live migrations in the Hyper-V settings of the VM.
- Add the IP addresses of both servers.
- Ensure that the firewall allows this service.
- Create a folder on WS1 to transfer the VM to.
- Restart WS1 and choose the "Move" option, selecting the option to move to a single location.
Step 9: Create a VM Template Using Sys Prep

Create a VM template using Sys Prep utility.

- Install the Sys Prep utility (out-of-box experience), then reboot the system.
- Export the VM in Hyper-V Manager and save the template to a specific location.
- Create a new VM based on the exported template.
