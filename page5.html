  5. Fun Guide to Setting Up DNS Server

    üåü Fun Guide to Setting Up Your DNS Server! üåü

    Step 1: Check the DNS Server Role

    1. Open the Server Manager on your Windows Server.
    2. Navigate to the Manage menu at the top right and select Add Roles and Features.
    3. In the Add Roles and Features Wizard, click Next to proceed.

    Did you know? The DNS Server translates domain names into IP addresses, making the internet user-friendly! üåê

    Step 2: Select the DNS Server Role

    1. On the Select Server Roles page, find and check the box for DNS Server.
    2. Click Next to continue through the wizard.

    Step 3: Complete the Wizard

    1. Review the features and settings for the DNS Server. Click Next until you reach the Confirm installation selections page.
    2. Finally, click Install to set up the DNS Server role. üéâ

    Step 4: Configure DNS as Secondary

    1. Once the installation is complete, open the DNS Management Console.
    2. Right-click on Forwarders and select Add Secondary Zone.
    3. Follow the wizard to create a new secondary zone for your domain:
      • Zone Type: Choose Secondary Zone and click Next.
      • Zone Name: Enter the name of the zone (e.g., yourdomain.com).
      • Master DNS Servers: Add the IP address of the primary DNS server.
    4. Click Finish to complete the setup of the secondary DNS zone.

    Step 5: Verify Configuration

    1. Check that the new secondary zone appears under the Zones section in the DNS Management Console.
    2. Perform DNS lookups to verify that the secondary server is resolving queries correctly.

    üéä Conclusion üéä

    By following these steps, you will successfully check the DNS Server role, install it, and configure it as a secondary DNS server for your domain. Happy Networking! üöÄ

      6. Join Windows Client VM to Domain

      üåü How to Join Your Windows Client VM to the Domain! üåü

      Step 1: Open System Properties

      1. Log in to your Windows client VM.
      2. Right-click on This PC (or My Computer) and select Properties.
      3. In the System window, click on Advanced system settings on the left.

      Step 2: Access Computer Name Settings

      1. In the System Properties window, click the Computer Name tab.
      2. Click on the Change... button to modify the computer's name or join a domain.

      Fun Fact: Each computer in a domain must have a unique name to avoid confusion! üñ•Ô∏è

      Step 3: Join the Domain

      1. Select the option Domain under Member of.
      2. Enter the name of the domain you want to join (e.g., yourdomain.com).
      3. Click OK to proceed.

      Step 4: Enter Domain Credentials

      1. When prompted, enter the credentials of a user account that has permission to join the domain.
      2. Click OK to continue.

      Step 5: Restart the VM

      1. You will see a welcome message confirming that your computer has been added to the domain.
      2. Click OK and then Close the System Properties window.
      3. Restart your Windows client VM to complete the process.

      üéä Conclusion üéä

      Congratulations! You've successfully joined your Windows client VM to the domain. Now you're all set to enjoy the benefits of centralized management and resources! üöÄ

      Creating Two-Way Forest Trusts

      üåü Creating Two-Way Forest Trusts! üåü

      Step 1: Open Active Directory Domains and Trusts

      1. Log in to the domain controller for the first forest.
      2. Open the Active Directory Domains and Trusts tool from the Administrative Tools.

      Step 2: Create a New Trust

      1. Right-click on the domain node and select Properties.
      2. Go to the Trusts tab.
      3. Click on New Trust... to launch the Trust Wizard.

      Step 3: Configure Trust Type

      1. Follow the wizard's prompts to enter the name of the second forest.
      2. Select Forest Trust and click Next.
      3. Choose Two-way trust to allow mutual access to resources.

      Did you know? Two-way trusts allow users in both forests to access resources in either domain! üåê

      Step 4: Set Authentication Options

      1. Choose the authentication type you want for the trust:
        • Forest-wide authentication allows users from either forest to access resources.
        • Selective authentication lets you control which users can access resources.
      2. Click Next and complete the wizard.

      Step 5: Validate the Trust

      1. After creating the trust, validate it by clicking the Validate button in the Trusts tab.
      2. Follow the prompts to confirm the trust relationship is established successfully.

      üéä Conclusion üéä

      Congratulations! You've successfully created a two-way forest trust, enabling seamless resource sharing between the two forests. üéâ
