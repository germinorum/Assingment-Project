Overview

Active Directory (AD) is a directory service developed by Microsoft for Windows domain networks. It is used for managing permissions and access to network resources.

Section 3: Active Directory

Goals

  • Understand the structure of Active Directory.
  • Implement best practices for group management.
  • Explore the benefits of using Active Directory in enterprise environments.

Key Features

  • Centralized Resource Management: Manage users, groups, and computers from a central location.
  • Security: Implement security policies and enforce user access permissions.
  • Scalability: Easily scale the directory as your organization grows.
  • Group Policies: Deploy settings and configurations across all computers in the domain.

Benefits of Active Directory

  • Improved security through centralized control.
  • Enhanced resource management and organization.
  • Streamlined user access and authentication.

Implementation Process

üåü Implementation Process üåü

Steps to Follow:

  1. Use BPA Scan on WS1:
    • Open the Server Manager.
    • Right-click on the server name (WS1) and select Best Practices Analyzer from the context menu.
    • Choose Start BPA Scan from the drop-down options.
    • After the scan, review the information displayed, noting any recommendations or issues highlighted.
  2. Delete Child Domain from WS2:
    • Access the Active Directory Users and Computers console.
    • Navigate to the Domain where the child domain resides.
    • Right-click on the child domain and select Delete.
    • Confirm the deletion when prompted.
  3. Use Windows Admin Center (WAC) on WS2:
    • Search for "Windows Admin Center" on Google, download, and install the latest version.
    • Open WAC after installation.
    • Click Settings and select Certificates to generate a self-signed certificate.
    • Use default port number 443 for secure access.
    • Note the URL provided to access WAC (e.g., https://:443).
    • Access WAC from a client web browser using the noted URL.
    • Add the server name WS1 by clicking on Add Server.
    • Create a new folder in WS1: right-click in the desired location and select New Folder.
    • Download a file from WS2 to WS1 using the Download option.
    • To restart the DHCP service on WS1, navigate to Services, find DHCP, right-click, and select Restart.
    • Explore menu items like Firewall, Network, Roles, Updates, etc., and note what configurations can be adjusted.
  4. Remove WAC Role from WS2:
    • Confirm that WAC is functioning correctly, then navigate to Server Manager.
    • Right-click on WS2, select Remove Roles and Features, and follow the wizard to remove WAC.
  5. Make WS2 a Second Domain Controller:
    • In Server Manager, right-click on AD DS and select Add Domain Controller.
    • Follow the prompts to join WS2 to the dileep1 domain.
  6. Ensure AD Replication:
    • Confirm that replication occurs between domain controllers every 180 minutes using the Active Directory Sites and Services console.
  7. Remove WS2 Global Catalog Ability:
    • In the AD Sites and Services console, right-click on WS2 and select Properties.
    • Uncheck Global Catalog and click OK.
  8. Check Functional Levels:
    • Open Active Directory Domains and Trusts from Server Manager.
    • Right-click on the domain and select Properties to check the current functional levels.
  9. Use AD Sites and Services:
    • In AD Sites and Services, remove the global catalog from WS2, ensuring it remains enabled for WS1.
  10. Active Directory Users and Computers:
    • Verify that client computers are added to the domain by checking the Computers container.
    • Confirm WS1 and WS2 are listed as domain controllers for the same domain.
    • Configure AD OUs, groups, and users on WS2 as follows:
      • OU Staff: Create a Universal user that can log into both domains.
      • OU Developers: Create a group named Developers.
      • OU Managers: Create a group named Managers.
      • OU Accounts: Create a group named Accounts.
  11. Restrict Developer User Access:
    • Set restrictions for the Developer user to allow access only on weekdays through User Properties.
  12. Test User Logins:
    • Ensure users can log in correctly from the client node.
  13. Create Checkpoint of WS1 and WS2:
    • In VMware, right-click on WS1 and select Take Snapshot.
    • Repeat for WS2 to create a checkpoint for both servers.
