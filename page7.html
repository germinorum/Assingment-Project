Section 4: File, Folders and DFS

🌟 Section 4: File, Folders and DFS 🌟

Goals

  • Assign AD group rights/permissions to folders.
  • Specify a file attribute.
  • Create a Distributed File System (DFS).
  • Create shared folders (SMB, NFS, and published in AD).
  • Create and apply quota and file screen restriction templates.
  • Configure data deduplication.
  • Set up the Windows Server Backup utility according to a daily schedule.

Implementation Steps for File and Folders

🚀 Implementation Steps for File and Folders 🌟

Let's Get Started!

  • 1. 🗂️ Create a companyDocs folder on C drive of WS2.
    • 🖱️ Right-click on the C drive, select New, then click Folder.
    • ✍️ Name it companyDocs and hit Enter.
  • 2. 📁 Inside companyDocs, create three folders:
    • 👨‍💻 Right-click companyDocs, select New, then Folder for Developers.
    • 👩‍💼 Repeat for Managers and Accounts folders.
    • 📄 Open each folder and create a file (e.g., a document) inside.
  • 3. 🔒 Assign permissions for each folder:
    • 🖱️ Right-click on the Developers folder, select Properties.
    • Click on the Security tab, then Edit.
    • 🔽 Select the Developers group, check Full Control, and click OK.
    • 🌐 Repeat for other folders with their respective groups.
  • 4. 🔑 Enable the ‘encrypt’ attribute for the Managers and Accounts folders.
    • 🖱️ Right-click on the Managers folder, select Properties.
    • Click on Advanced, check Encrypt contents to secure data, and click OK.
    • 🔁 Repeat for the Accounts folder.
  • 5. 🖥️ Log in as each user on the Windows client and test access to the folders.
    • 🔄 Switch user and log in as Developers, Managers, and Accounts.
    • 📂 Open Windows Explorer and navigate to the folders. Verify access permissions!
  • 6. 🌐 Create a Distributed File System (DFS):
    • 🖱️ Right-click on the DFS Management console, select New Namespace.
    • 🔽 Add WS1 and WS2 as namespace servers.
    • 🗂️ Create a target pointing to the companyDocs folder.
    • 📁 Create a folder on WS2 called companyDocs Replica.
    • 🔄 Replicate the DFS namespace on both servers.
    • 🧪 Test that folders created locally on both servers are replicated on the other!
  • 7. 📤 Create three shared folders on WS1:
    • 🖱️ Right-click the Server Manager, go to File and Storage Services.
    • 🔽 Choose Shares, then Tasks and New Share.
    • 📝 Follow the wizard for SMB (WDS images), NFS (LinuxFiles), and AD published folder (Applications).
    • ❓ Learn about sharing methods along the way!
    • 🔍 Access shared folders from a network client.
  • 8. 🚫 Configure restrictions:
    • 🖱️ Right-click on the Developers folder, go to Properties.
    • 🔽 Select the Quota tab and set it to 1GB.
    • 📁 For the Accounts folder, set up a file screen for audio, video, and other file types.
  • 9. 💾 Configure data deduplication:
    • 🖱️ In VM settings, Add a virtual hard drive.
    • 🔽 Create a new volume E: and a folder called dataDedup.
    • ⚙️ Enable data deduplication from the Server Manager.
  • 10. ⏳ Set up Windows Server backup:
    • 🗂️ Create a backup folder on WS2: shareFoldersWindowsBackup.
    • 🖱️ In Backup, select Custom Backup of C drive.
    • ⚡ Choose Fast Performance and Schedule backups accordingly.
