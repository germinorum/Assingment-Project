Section 5: Remote Access and Monitoring

Goals

  • Configure and use Remote Desktop (RD) Session-based desktop deployment on WS2
  • Configure and use RD web access on WS2
  • Configure and use RD connection broker for WS2
  • Restrict RD access to specific AD groups
  • Make WS1 and RD gateway to R2 and connect from Rexnet/Wintec computer
  • Use the following built-in server monitoring tools:
    • Performance Monitor
    • Task Manager
    • Resource Manager
    • Event Viewer
    • Reliability Monitor
    • Set local server performance alerts

Implementation Steps

1. Ensure WS1 has two network adapters.

One connected to the Rexnet network and one connected to your internal network. It should also function as a domain controller and IIS web server.

2. Create a shared folder called certificates on WS1.

This folder should be accessible by all users in the domain and can also be accessed from WS2.

3. Add the Remote Desktop service on WS2.

Note: This is the service, not the role. Choose standard deployment and enable RD web access, session-based desktop deployment, and connection broker.

4. Add servers: WS1 to WS2 and vice versa.

This allows both servers to communicate with each other effectively.

5. Add the RD gateway role to WS1.

Specify WS1 as the RD gateway and the SSL certificate name: WS1_hostname.your_domain. The installation may take time, but the role will appear on WS1 when complete.

6. Create a self-signed certificate.

Use the name from the previous step and save it in the shared folder, naming it the same as the FQDN of WS1.

7. Create CAP and WAP policies.

These policies restrict access, allowing only Manager and Accounts users to use RD to access WS2.

8. Use the remote desktop application on a random Rexnet/wireless client.

Connect to WS2 through the RD Gateway. Use the FQDN of WS1 for RD gateway settings, matching the certificate you created.

9. Access WS2 from an internal client using RD web.

Add RDApp programs accessible via the internet, such as Calculator and Server Manager.

10. Use server monitoring tools on both servers.

Take screenshots of the following:

  • Task Manager and Resource Manager: Identify applications and users consuming the most resources.
  • Performance Monitor: Add three features to monitor and collect system diagnostic data.
  • Event Viewer: Check for errors related to the data collection set.
  • Reliability Monitor: Review the average rating for your server over the last week.
  • Set local server performance alerts: Choose two alerts and explain your selections.
